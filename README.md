# Git工作流助手

一个强大的 Git 工作流自动化 VSCode 插件，集成了快速创建标准化分支和自动化分支合并流程两大核心功能，采用模块化架构设计，让您的 Git 工作流更加高效、安全、智能。

## ✨ 功能特性

### 分支创建功能
- 🎯 **快速创建分支**：一键创建标准化命名的 Git 分支
- 📋 **规范统一**：统一团队分支命名规范（前缀/日期/描述_用户名）
- 🔧 **灵活配置**：支持自定义分支前缀和命名格式
- 👤 **自动识别**：自动获取 Git 用户名或使用自定义名称
- 📅 **日期格式**：支持多种日期格式（yyyyMMdd、yyyy-MM-dd、yyMMdd）
- 🔀 **基分支选择**：可选择任意本地或远程分支作为基础分支
- ✅ **输入验证**：完整的输入验证，确保分支名称合法
- 💾 **状态栏快捷**：状态栏一键访问创建分支功能

### 分支合并功能
- 🚀 **自动化合并流程**：一键完成 Feature 分支到目标分支的合并
- ⚡ **快速提交合并**：快速提交当前更改并合并到目标分支
- 🎯 **智能分支检测**：支持多种分支命名模式（feature、feat、bugfix、hotfix、fix）
- 🔧 **灵活配置管理**：支持自定义主分支和目标分支配置
- 🛡️ **合并冲突处理**：智能检测和处理合并冲突，提供多种解决方案
- 🔒 **并发控制**：防止多个合并操作同时执行
- 🌐 **远程分支验证**：自动验证远程分支存在性
- 📊 **环境验证**：全面的 Git 环境检查和状态监控

### 通用特性
- 🔄 **多种调用方式**：支持命令面板、右键菜单、SCM 面板、状态栏等多种方式
- 📝 **友好提示**：详细的操作提示和错误信息
- 🏗️ **模块化架构**：清晰的代码结构，易于维护和扩展
- ⚡ **高性能**：轻量级实现，响应迅速

## 🚀 快速开始

### 使用方法

#### 方法一：命令面板
1. 按 `Ctrl+Shift+P`（Mac: `Cmd+Shift+P`）打开命令面板
2. 输入 "Git工作流助手" 或相关命令名称
3. 选择相应的命令执行：
   - **创建功能分支**：快速创建标准化命名的分支
   - **管理分支前缀**：添加、删除或设置默认分支前缀
   - **合并Feature分支**：执行完整的分支合并流程
   - **快速提交并合并**：快速提交更改并执行合并
   - **配置管理**：管理插件配置

#### 方法二：快捷键
- **创建分支**：`Ctrl+Alt+Shift+B`（Mac: `Cmd+Alt+Shift+B`）
- **合并分支**：`Ctrl+Alt+Shift+M`（Mac: `Cmd+Alt+Shift+M`）

#### 方法三：状态栏
- 点击左下角状态栏的 "创建分支" 按钮快速创建分支

#### 方法四：右键菜单
- 在资源管理器中右键文件夹
- 在编辑器中右键

#### 方法五：SCM面板
- 打开源代码管理面板
- 点击标题栏中的相应按钮

## 📦 主要功能

### 一、创建功能分支

快速创建符合团队规范的标准化分支名称。

#### 操作流程
1. **触发创建**：通过快捷键、命令面板或状态栏触发
2. **选择前缀**：从配置的前缀列表中选择（如 feature、bugfix、hotfix）
3. **选择基分支**：选择要基于的分支（默认显示当前分支）
4. **输入描述**：输入分支的描述信息（支持中英文、数字、下划线、短横线）
5. **预览确认**：查看生成的分支名称并确认
6. **自动创建**：自动创建分支并可选择是否切换

#### 分支命名格式
```
前缀/日期/描述信息_用户名
```

#### 使用示例
```
# 示例 1: 创建功能分支
feature/20250101/用户登录功能_john

# 示例 2: 创建修复分支
bugfix/20250101/修复登录bug_mary

# 示例 3: 创建热修复分支
hotfix/20250101/紧急修复支付异常_alex
```

#### 特性说明
- **自动获取用户名**：从 Git 配置自动读取，也可在设置中自定义
- **智能验证**：实时验证分支名称的合法性
- **重复检测**：自动检测分支是否已存在，提供切换或重新输入选项
- **灵活配置**：支持多种日期格式和自定义前缀

### 二、合并Feature分支
采用重构后的模块化流程：
1. **环境验证**：全面检查Git仓库状态和配置完整性
2. **分支验证**：智能检查当前分支是否为功能分支
3. **冲突预检**：检查未提交的更改和潜在冲突
4. **主分支更新**：安全切换到主分支并拉取最新代码
5. **智能合并**：带冲突检测的安全合并机制
6. **冲突处理**：如有冲突，提供多种解决方案
   - 打开冲突文件在编辑器中编辑
   - 中止合并操作并回滚
   - 等待用户手动解决后继续
7. **自动推送**：合并成功后自动推送结果到远程

### 三、快速提交并合并
优化的快速操作流程：
1. **变更检测**：自动检测未提交的更改
2. **输入验证**：验证提交信息格式和长度
3. **自动暂存**：暂存所有更改并提交
4. **合并确认**：询问是否继续执行合并流程
5. **流程集成**：无缝集成到完整合并流程

### 四、配置管理
全面的配置管理系统：
1. **主分支设置**：支持预设选项和自定义输入
   - master、main、release、develop预设选项
   - 自定义分支名称输入
   - 远程分支存在性验证
2. **目标分支管理**：动态管理目标分支列表
   - 添加新的目标分支
   - 删除现有目标分支
   - 分支描述和验证
3. **功能分支模式**：配置分支命名规则
   - 添加新的分支模式
   - 删除现有模式
   - 模式有效性验证
4. **自动检测开关**：控制主分支自动检测
5. **配置重置**：支持选择性重置配置
   - 重置所有配置
   - 重置特定配置项
   - 重置确认和恢复

### 五、分支前缀管理

专门用于管理分支创建时使用的前缀。

#### 功能包括
1. **添加新前缀**：添加自定义的分支前缀

## ⚙️ 配置选项

### 分支合并配置

#### 主分支配置
- **gitWorkflowHelper.mainBranch**：设置主分支名称（支持任意自定义名称）
- **gitWorkflowHelper.autoDetectMainBranch**：是否自动检测主分支（默认关闭）

#### 目标分支配置
- **gitWorkflowHelper.targetBranches**：配置可选的目标分支列表

#### 功能分支配置
- **gitWorkflowHelper.featureBranchConfig**：配置功能分支命名规则

### 分支创建配置

#### 分支前缀配置
- **gitWorkflowHelper.branchPrefixes**：配置分支前缀列表（用于创建分支）

#### 用户名配置
- **gitWorkflowHelper.customGitName**：自定义 Git 用户名（留空则使用 Git 配置）

#### 日期格式配置
- **gitWorkflowHelper.dateFormat**：分支命名中的日期格式
  - `yyyyMMdd`：20250101（默认）
  - `yyyy-MM-dd`：2025-01-01
  - `yyMMdd`：250101

#### 自动切换配置
- **gitWorkflowHelper.autoCheckout**：创建分支后是否自动切换（默认开启）

### 完整配置示例

```json
{
  "gitWorkflowHelper.mainBranch": "master",
  "gitWorkflowHelper.autoDetectMainBranch": false,
  "gitWorkflowHelper.targetBranches": [
    {
      "name": "uat",
      "description": "测试环境"
    },
    {
      "name": "pre",
      "description": "预发布环境"
    }
  ],
  "gitWorkflowHelper.featureBranchConfig": {
    "patterns": ["feature", "feat", "bugfix", "hotfix", "fix"],
    "description": "功能分支命名模式"
  },
  "gitWorkflowHelper.branchPrefixes": [
    {
      "prefix": "feature",
      "description": "功能分支",
      "isDefault": true
    },
    {
      "prefix": "bugfix",
      "description": "修复分支",
      "isDefault": false
    },
    {
      "prefix": "hotfix",
      "description": "热修复分支",
      "isDefault": false
    }
  ],
  "gitWorkflowHelper.customGitName": "",
  "gitWorkflowHelper.dateFormat": "yyyyMMdd",
  "gitWorkflowHelper.autoCheckout": true
}
```

## 🔧 故障排除

### 常见问题

#### 合并冲突
当遇到合并冲突时，插件会：
1. **自动检测冲突文件**：识别所有冲突状态码（UU, AA, DD等）
2. **提供处理选项**：
   - **打开冲突文件**：在编辑器中打开第一个冲突文件
   - **中止合并**：取消当前合并操作并回滚
   - **手动解决后继续**：等待用户解决冲突后继续
3. **智能等待机制**：监控冲突解决状态
4. **自动提交**：冲突解决后自动提交合并结果

#### 分支检测失败
如果当前分支未被识别为功能分支：
1. **检查分支名**：确认分支名包含配置的模式
2. **添加新模式**：通过配置管理添加新的分支模式
3. **验证配置**：确认功能分支配置正确
4. **团队规范**：确保分支命名符合团队规范

#### 远程分支不存在
如果配置的主分支在远程不存在：
1. **检查连接**：验证远程仓库连接状态
2. **确认名称**：检查分支名称拼写正确
3. **使用自动检测**：开启主分支自动检测功能
4. **手动配置**：通过配置管理设置正确的主分支名

#### 环境验证失败
使用新增的环境验证功能：
1. **运行验证**：通过配置管理执行环境验证
2. **查看报告**：检查详细的验证报告
3. **修复问题**：根据报告修复配置问题
4. **重新验证**：确认所有问题已解决
